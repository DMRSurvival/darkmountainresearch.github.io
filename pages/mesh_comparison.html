Review of Wireless Mesh Technologies

Meshtastic

Meshtastic is an open-source, decentralized, off-grid mesh network designed to run on affordable, low-power devices. It primarily utilizes LoRa (Long Range) radio technology, a spread spectrum modulation technique that allows for long-range communication with minimal power consumption [1], [2]. The system is designed for scenarios where internet access is unavailable or unreliable, enabling users to send text messages and location data through a network of devices that relay messages for one another.

  • Platform & Connectivity: Meshtastic operates on low-power hardware (such as ESP32 or nRF52 based boards) and offers client applications for iOS, Android, and web browsers to manage the mesh and send messages [1].
  • Protocol: It uses a mesh protocol over LoRa, which supports ranges of up to 10 miles (16 km) in rural areas, though practical ranges vary by environment [1], [3].
  • Use Case: Ideally suited for outdoor activities (hiking, skiing), emergency situations, and hobbyist off-grid communications.

MeshCore

MeshCore is a multi-platform system focused on enabling secure, text-based communications using LoRa radio hardware. Like Meshtastic, it functions as an off-line and off-grid messaging platform that does not rely on the internet [4].

  • Mission: Its stated mission is to create a reliable, secure, and decentralized mesh radio networking system accessible to anyone.
  • Target Audience: It is tailored for emergency response, disaster recovery, tactical security (law enforcement and private security), and outdoor activities.
  • Key Features: The system emphasizes security and efficiency, supporting both human communication and IoT sensor networks.

AREDN (Amateur Radio Emergency Data Network)

AREDN is a high-speed data network implementation that repurposes commercial wireless hardware (such as routers from Ubiquiti, Mikrotik, and TP-Link) for use in the Amateur Radio bands [5]. Unlike the low-bandwidth LoRa systems, AREDN is capable of supporting higher bandwidth applications typical of standard TCP/IP networks.

  • Routing Protocol: AREDN recently transitioned from the legacy OLSR (Optimized Link State Routing) protocol to Babel, a modern routing protocol. As of release 4.26.1.0, the firmware is "Babel only," addressing scalability issues that OLSR faced in large networks [6], [7].
  • Spectrum: It operates on amateur radio frequencies, including 2.4 GHz, 5 GHz, and increasingly 900 MHz via IEEE 802.11ah (Wi-Fi HaLow) hardware supported in newer builds [8], [9].
  • Architecture: It provides a tunnel backup/restore feature and runs on top of the OpenWrt operating system (specifically OpenWrt 24.10.5 in recent versions) [8].

Open Mesh (B.A.T.M.A.N. Advanced)

"Open Mesh" in this context refers to the Open Mesh project and community that develops the B.A.T.M.A.N. (Better Approach to Mobile Ad-hoc Networking) routing protocol [10]. Specifically, batman-adv is a Linux kernel module that operates at the Data Link Layer (Layer 2) rather than the Network Layer (Layer 3) [11], [12].

  • Layer 2 Virtual Switch: Unlike AREDNâ€™s Babel (Layer 3), batman-adv encapsulates and forwards raw Ethernet frames, emulating a giant virtual network switch across all participating nodes. This makes the mesh transparent to higher-level protocols; nodes appear to be link-local, allowing DHCP, IPv6, and non-IP protocols to function without special configuration [13], [14].
  • Decentralized Routing: The protocol decentralizes routing knowledge so no single node possesses the entire network map, which improves scalability and eliminates the need to propagate all topology changes to every node [15].
  • Implementations (OpenMANET): Projects like OpenMANET utilize batman-adv on Raspberry Pi hardware combined with Wi-Fi HaLow (802.11ah) radios to create long-range, self-forming mesh networks [16], [17]. These implementations leverage the "virtual switch" capability to bridge traffic over sub-GHz frequencies.

Reticulum

Reticulum is a cryptography-based networking stack designed to build "unstoppable networks" that are resistant to surveillance, censorship, and control [18]. It is distinct because it is not a single network but a tool for building many interoperable, autonomous networks.

  • Transport Agnostic: Reticulum can operate over almost any medium, including LoRa, Packet Radio, Wi-Fi, Ethernet, and I2P. It is capable of functioning in conditions with very high latency and extremely low bandwidth [18].
  • Privacy and Security: It does not use source addresses; packets do not contain information about their origin. All communication is encrypted by default with ephemeral keys, offering forward secrecy [19].
  • Addressing: Addresses are "self-sovereign" and portable, meaning they can be moved physically to different locations in the network without losing reachability, and there is no central authority controlling address allocation [19].
